## Critical Learning: async tests required for backit module

**ALL tests for classes defined in the `backit` module that conform to protocols with `async`
methods MUST be `async` test methods.**

Root cause: `backitApp: App` (SwiftUI, implicitly `@MainActor`) is in the `backit` module.
Swift 5.9+ applies global actor inference, causing classes with async methods (CCCJob, DropboxJob,
and anything else in `backit` with async protocol requirements) to get executor-aware deinit via
`swift_task_deinitOnExecutorImpl`. Synchronous XCTest methods have no Task context, causing
`TaskLocal::StopLookupScope::~StopLookupScope()` to call `free()` on a static sentinel address.

**Fix pattern:**
```swift
// WRONG ‚Äî crashes with malloc error for backit-module classes with async protocol methods
func testFoo() { let job = SomeJob(); XCTAssertEqual(...) }

// CORRECT
func testFoo() async { let job = SomeJob(); XCTAssertEqual(...) }
```

This applies to: CCCJobTests ‚úÖ fixed, DropboxJobTests ‚úÖ fixed.
Will also apply to: BackupCoordinatorTests, any future job/coordinator tests.
